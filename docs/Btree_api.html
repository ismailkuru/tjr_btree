Module Btree_api

module Btree_api: sig .. end
B-tree api types

This module describes the main interfaces. The interfaces are heavily parameterized.

To understand the interfaces, we need to introduce the following:

  • Keys, represented by type variable 'k
  • Values, by type var 'v
  • Page/block references, 'r
  • Global state, 't
The operations typically execute in the Base_types.Monad.
type 't disk_ops = {
   block_size : Default.BLK.sz;
   read : Default.BLK.r -> (Default.BLK.t, 't) Base_types.m;
   write : Default.BLK.r -> Default.BLK.t -> (unit, 't) Base_types.m;
   disk_sync : unit -> (unit, 't) Base_types.m;
}
Disk operations: read, write, and sync
type ('k, 'v, 'r, 't) store_ops = ('k, 'v, 'r, 't) Base_types.Store_ops.store_ops = {
   store_free : 'r list -> (unit, 't) Base_types.m;
   store_read : 'r -> (('k, 'v, 'r) Base_types.frame, 't) Base_types.m;
   store_alloc : ('k, 'v, 'r) Base_types.frame -> ('r, 't) Base_types.m;
}
Store operations: alloc, free and read. The store is a level above the raw disk, on which we build the B-tree.

Leaf stream representation. This type is for debugging - you shouldn't need to access components.
type ('k, 'v, 'r, 't) ls_ops = {
   mk_leaf_stream : unit -> (('k, 'v, 'r) lss, 't) Base_types.m;
   ls_step : ('k, 'v, 'r) lss ->
(('k, 'v, 'r) lss option, 't) Base_types.m;
   ls_kvs : ('k, 'v, 'r) lss -> ('k * 'v) list;
}
A leaf stream is a linear sequence of the leaves in a B-tree, used for iterating over all the bindings in the tree. Leaf stream operations: make a leaf stream; get the list of (key,value) pairs associated to the state of the leaf stream; step to the next leaf.
val all_kvs : ('k, 'v, 'r, 't) ls_ops -> (('k * 'v) list, 't) Base_types.m
Get all (key,value) pairs from a leaf stream. Debugging only.
